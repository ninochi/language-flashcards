# Learning Design

このアプリの目的は、`data/` 以下のJSONに入っている既知の単語集合を、追加生成せずに強く記憶することです。

## 方針

- デッキを「今日のノルマ」として有限化しない。
- JSON内の単語集合を出題アルゴリズムの母集団として扱う。
- ユーザーが飽きたところで自然に終われる軽さを保つ。
- 双方向出題や、漢字・拼音・意味の分解モードは今は入れない。
- 達成演出は「完全習得」ではなく、学習マイルストーンとして扱う。

## 回答区分

回答は次の3段階にする。

- `分からなかった`: 答えを見るまで出なかった。
- `迷った`: 一部出た、時間がかかった、声調・拼音・綴りなどに不安がある。
- `すぐ分かった`: 答えを見る前に、すぐ確実に思い出せた。

`簡単` ではなく `すぐ分かった` と表現する。単語そのものの難易度ではなく、その場でどれだけ速く確実に取り出せたかを評価するため。

## SRSへの反映

- `分からなかった`: streakをリセットし、同じセッション内では十分に間を空ける。残りカードが少ない場合は無理に即再出題せず、次回以降の復習へ回す。
- `迷った`: 弱点として記録し、通常より短い間隔で復習する。
- `すぐ分かった`: streakを伸ばし、通常の復習間隔で次回復習を設定する。

通常の復習間隔は `1, 3, 7, 14, 30, 60` 日とする。

## 出題優先度

出題順は、単純シャッフルではなく優先度付きキューで決める。

1. セッション内で十分に間を空けた `分からなかった` カード
2. 復習期限が来たカード
3. 最近 `迷った` カード
4. 未出題カード
5. 長く見ていないカード
6. ランダム補充

この優先度は、学習を制限するノルマではなく、次に出す価値が高いカードを自然に選ぶためのもの。キューを一通り終えたあとも、ユーザーが続けたい場合はもう一周できる。

## マイルストーン演出

confettiなどの演出は、次のようなマイルストーンに使う。

- そのデッキの全カードを一度見た。
- そのデッキの全カードを一度以上 `すぐ分かった` にした。
- 復習期限が来ていたカードを一通り処理した。

表示文では `完全習得` とは言わず、`このデッキを一周しました` のように事実ベースで表現する。

現時点で実装する演出は次の2つ。

- `このデッキを一周しました`: 全カードを一度見たとき。
- `全カードを一度すぐ思い出せました`: 全カードで一度以上 `すぐ分かった` を選んだとき。

どちらもデッキごとに一度だけ表示する。学習記録をリセットした場合は、マイルストーンもリセットする。

confettiは、最後のカードの次を表示したタイミングではなく、マイルストーンを満たした回答を選んだタイミングで表示する。
