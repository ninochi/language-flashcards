# Learning Design

このアプリの目的は、`data/` 以下のJSONに入っている既知の単語集合を、追加生成せずに強く記憶することです。

## 方針

- 忘却曲線や日付ベースのSRSは使わない。
- 回答区分は `分からなかった` と `分かった` の2つだけにする。
- `分からなかった` カードをデッキごとにブラウザへ保存する。
- `分からなかっただけ` モードでは、そのリストだけを回して、分かったものから減らしていく。
- 進捗は端末の `localStorage` に保存し、デッキごとに分離する。
- 達成演出は、`分からなかった` がゼロになった時だけ表示する。
- クイズは短時間で終われる20問セッションにする。

## カード学習

カード学習は、日本語を見て裏面の台湾語ローマ字または中国語を思い出す。

- `分からなかった`: 答えを見るまで出なかった。カードIDを `unknownIds` に入れる。
- `分かった`: 答えを見る前に思い出せた。カードIDを `unknownIds` から外す。

`カード` モードでは全カードを出題する。すでに `unknownIds` に入っているカードには `分からなかった` ラベルを表示する。

`分からなかっただけ` モードでは、`unknownIds` に入っているカードだけを出題する。このモードで `分かった` を選ぶとリストから外れるため、残りが減っていく。`分からなかった` を選んだ場合は、まだ覚えていないものとして同じモードの末尾へ戻す。

## クイズ

クイズモードは、該当デッキからランダムに選んだ20問で、各カードの裏面を正解にした4択クイズを出す。デッキが20問未満の場合は、そのデッキの全カードを出す。

- セッション開始時に該当デッキから最大20問をランダム抽出する。
- 選択肢は同じデッキ内の既存カードから自動生成する。
- 正解を1つ、他カードの裏面から不正解選択肢を3つ選ぶ。
- 選択肢の表示順は毎回ランダムにする。
- クイズ中のステータスは、そのセッションの `正解数` と `間違えた数` を表示する。
- 間違えたカードIDは `quizWrongIds` に保存する。
- 次回以降、`quizWrongIds` に入っている問題には `前回間違えた問題` ラベルを表示する。
- 20問を解答後、全問正解ならconfettiを表示する。1問でも間違えた場合は残念表示にする。
- 結果画面の再挑戦ボタンを押すと、同じデッキから新しくランダム20問を抽出する。

4択候補は現時点ではJSONへ固定保存しない。将来、手動調整したい選択肢が出てきた場合は、カードデータへクイズ選択肢フィールドを追加する。

## 保存形式

保存キーは次の形式にする。

```text
language-flashcards:<storageKey>:v3
```

保存する状態は次の3種類を基本にする。

```json
{
  "version": 3,
  "unknownIds": ["c1-001"],
  "seenIds": ["c1-001", "c1-002"],
  "quizWrongIds": ["c1-010"]
}
```

`reset` は、そのデッキの上記状態を初期化する。単語JSON自体は変更しない。

## マイルストーン演出

カード学習のconfettiは、`unknownIds` が1件以上ある状態から0件になった時だけ表示する。

初回ロード時に `unknownIds` が0件でも演出しない。全カードを一周しただけでも演出しない。

クイズでは、20問セッションを全問正解した結果画面でconfettiを表示する。過去の `quizWrongIds` がゼロになっただけでは表示しない。
